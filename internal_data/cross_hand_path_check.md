# 跨手路径检查

来源：

- `internal_data/manual_key_layout.json`
- `internal_data/shouyin_constraint_priority.md`

目的：检查当前首音区与右手核心乐音区之间的跨手路径是否自然，识别当前布局里最顺与最别扭的组合。

## 方法

本检查使用一个简化的键位坐标模型：

- 按键按标准键盘行列近似映射到二维坐标。
- 路径代价用曼哈顿距离近似。
- 只检查高优先首音与其主要候选乐音，不评估全部音节。

这是一个粗略布局指标，不代表真实手感，但足够用于比较当前草案中的相对好坏。

## 一级首音结果

检查对象：`N19 N20 N21`

主要候选乐音：`M01 M03 M07 M09 M02 M08`

当前位置：

- `N19` -> `a/shift`
- `N20` -> `s/shift`
- `N21` -> `d/shift`

当前平均距离：

1. `N19`：`8.12`
2. `N20`：`7.12`
3. `N21`：`6.12`

结论：

- 在当前 `a/s/d/f` 带中，越靠右的成员，通往右手核心乐音区越顺。
- `N21` 明显优于 `N19`。
- 这说明 `G2-3` 当前的总体方向是对的：高约束首音组必须尽量靠近中缝。

## 二级首音结果

检查对象：`N09 N10 N13 N18 N23`

主要结果：

1. `N18` 平均距离：`2.75`
2. `N23` 平均距离：`2.21`
3. `N13` 平均距离：`4.46`
4. `N10` 平均距离：`5.71`
5. `N09` 平均距离：`6.96`

结论：

- 目前二级组里，已经贴近右手核心乐音区的，是 `N18` 与 `N23`。
- `N13` 也不差，说明 `G2-1` 的 `z/x/c/v` 方案并没有拖后腿。
- 真正偏远的是 `N09` 与 `N10`，也就是 `G1-3` 的两个实际软腭音。

## 当前最顺的部分

当前布局中，跨手路径最顺的三类关系是：

1. `N18` 到 `M10 M13 M15` 一带
2. `N23` 到 `M10 M16 M18` 一带
3. `N13` 到 `M10 M12 M13` 一带

这些关系之所以顺，是因为：

- 首音已经靠近左手中下区或中上区。
- 右手对应乐音又正好集中在 `f/j` 轴线附近的核心块。

## 当前最别扭的部分

当前布局中，最远的几类关系集中在：

1. `N19` 到 `M03/M09`
2. `N09` 到 `M06`
3. `N10` 到 `M06`

共性是：

- 首音偏左外侧。
- 乐音目标偏右上或右下外侧。
- 跨度大，而且没有中缝缓冲。

## 关于 `G2-3` 还能不能再挪

对 `G2-3` 做了一个简化对比：

- 若放左手上排连续带，平均距离约为 `6.88`
- 若放左手本排连续带，平均距离约为 `6.62`
- 若放左手下排连续带，平均距离约为 `6.12`

解释：

- 纯按几何距离算，`G2-3` 放得越低越靠近右手下排高元音组。
- 但如果把 `G2-3` 挪到下排，就会破坏当前已经成型的第二大组三层结构。
- 因此，当前 `a/s/d/f` 的方案不是绝对最短，但属于结构与路径之间的合理折中。

## 关于 `G1-3` 还能不能再挪

当前 `G1-3` 的问题不是“完全错误”，而是：

- 作为第一大组收束区，它放在左上外缘是合理的。
- 但其中的 `N09/N10` 到右手核心乐音区距离仍偏长。

因此，`G1-3` 还有可优化空间，但收益不会像前几轮回填那样大。

## 当前判断

如果按“路径质量”和“结构稳定性”综合判断：

1. 不建议再大动 `G2-1/G2-2/G2-3`。
2. 若还要继续调，优先只微调 `G1-3`。
3. 再往下，最值得做的不是继续搬键，而是建立一份练习顺序，让最自然的跨手路径先被习得。
