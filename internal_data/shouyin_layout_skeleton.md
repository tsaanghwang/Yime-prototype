# 首音布局设计依据（已同步到当前布局真源）

本文件记录当前布局中 21 个实首音（非零声母）、零首音及 `y/w` 的排键结果
和形成理由。乐音位置见 `internal_data/musical_layout_skeleton.md`。

当前实现以 `internal_data/manual_key_layout.json` 为唯一布局真源；
`internal_data/yinyuan_id_crosswalk.md` 是由它生成的核对表。
`internal_data/shouyin_group_template.json` 仍保留较早的分组资料，不能作为
当前实际布局依据。

## 上游乐音约束

47 个可分配的 Base 层物理键不包含 Space。已经固定的 22 个乐音键为：

```text
Top:          W=o低  E=o中  R=o高       U=u高  I=u中  O=u低
Home: A=n高  S=a低  D=a中  F=a高       J=i高  K=i中  L=i低  ;=ng高
Bottom: Z=n低 X=e低 C=e中  V=e高       M=ü高  ,=ü中  .=ü低  /=ng低
```

修改首音布局时，不得在没有同时调整乐音设计的情况下占用以上 22 个键。

## 21 个实首音

当前编号下，21 个实首音是 `N01–N11` 和 `N13–N22`，分为六小组。

### 第一大组

- G1-1 唇音组（4 个）：`N01 b` `N02 p` `N03 f` `N04 m`
- G1-2 龈音组（4 个）：`N05 d` `N06 t` `N07 l` `N08 n`
- G1-3 软腭组（3 个）：`N09 g` `N10 k` `N11 h`

### 第二大组

- G2-1 齿音组、平舌音组（3 个）：`N13 z` `N14 c` `N15 s`
- G2-2 龈腭间音组、翘舌音组（4 个）：`N16 zh` `N17 ch` `N18 sh` `N19 r`
- G2-3 硬腭前部组（3 个）：`N20 j` `N21 q` `N22 x`

六组规模依次为 `4、4、3、3、4、3`，合计 21 个。

## 21个实首音以外的成员

以下三个 Yinyuan ID 不属于传统的21个实首音：

- `N12`：零首音占位，使用物理键 `'`；
- `N23`：`y`，保留在 Base 层，使用物理键 `Y`；
- `N24`：`w`，保留在 Base 层，使用物理键 `=`。

`N23 y`、`N24 w` 虽不计入21个实首音，但实际使用频率较高，因此进入
Base 层。`N12` 使用原拼音隔音符号键 `'`，把“隔音符号”和“没有实首音”
合并成一条记忆规则。

## 布局重构时解决的冲突

重构前的可打方案有两个实首音键被新的乐音骨架占用：

| Yinyuan ID | 首音 | 重构前键位 | 新占用 | 当前处理 |
|---|---|---|---|---|
| `N05` | `d` | `W` | `/o/` 低音 `M15` | 改到 `]` |
| `N06` | `t` | `.` | `/ü/` 低音 `M09` | 改到 `T` |

`N06 t` 的最终位置为：

```text
N06 t = T（Base 层，VK_T）
```

随后为降低学习负担，确定一组同字母或重新锚定的位置：

```text
N01 b = B（Base 层，VK_B）
N23 y = Y（Base 层，VK_Y）
N08 n = N（Base 层，VK_N）
N09 g = G（Base 层，VK_G）
N10 k = Q（Base 层，VK_Q）
N11 h = H（Base 层，VK_H）
```

这些调整使原来位于 `H` 的 `N04 m` 和原来位于 `B` 的 `N07 l` 需要重新
定位；它们的最终试排位置见下表。

## 六个标点符号键的分配

固定22个乐音和其余首音后，六个标点符号键按两个相邻双键组及零首音规则
分配：

```text
`  = 保留
-  = N04 m
=  = N24 w
]  = N05 d
\  = N07 l
'  = N12 零首音
```

其中：

- `- =` 组成唇部相关的 `m/w` 双键组；
- `] \` 组成舌尖龈音 `d/l` 双键组；
- `'` 沿用拼音隔音符号的含义，表示零首音；
- `` ` `` 是当前唯一未分配的 Base 保留键。

21 个实首音、`y/w` 和零首音至此均已有 Base 键位。

## 候选选择

候选选择不再占用 Base 音元键：

```text
Space、Enter 或 Shift+1 = 提交首选
Shift+2                 = 第2候选
Shift+3                 = 第3候选
Shift+4                 = 第4候选
Shift+5                 = 第5候选
Shift+6                 = 第6候选
Shift+7                 = 第7候选
Shift+8                 = 第8候选
Shift+9                 = 第9候选
```

Shift 层必须预留 `Shift+1` 至 `Shift+9` 给候选选择。其余11个乐音的
Shift 键位也已经全部确定，具体安排以
`internal_data/musical_layout_skeleton.md` 为准。
