# 首音布局一致性检查

来源：

- `internal_data/manual_key_layout.json`
- `internal_data/shouyin_layout_skeleton.md`
- `internal_data/shouyin_group_template.json`
- `internal_data/shouyin_constraint_priority.md`

目的：检查当前结构版布局草案是否满足首音分组相邻关系与优先级要求。

## 当前结论

当前版 `manual_key_layout.json` 已经把六个首音小组全部收拢成可识别的组块，结构上比前几轮稳定很多。

当前状态可以概括为：

1. 第二大组三小组已经完整形成三条连续的左手 `shift` 带。
2. 第一大组中的 `G1-1`、`G1-2` 已经形成数字行 `base/shift` 对应双层块。
3. `G1-3` 已经收拢到左侧外缘 `base` 区，并且最外角已让给虚首音，但还不是最理想的最终形态。

### 第一大组

- `G1-1`：`N01 2/base` `N02 3/base` `N03 4/base` `N04 5/base`
- `G1-2`：`N05 2/shift` `N06 3/shift` `N07 4/shift` `N08 5/shift`
- `G1-3`：`N22 `/base` `N09 1/base` `N10 q/base` `N11 a/base`

### 第二大组

- `G2-1`：`N12 z/shift` `N13 x/shift` `N14 c/shift` `N23 v/shift`
- `G2-2`：`N15 q/shift` `N16 w/shift` `N17 e/shift` `N18 r/shift`
- `G2-3`：`N19 a/shift` `N20 s/shift` `N21 d/shift` `N24 f/shift`

## 已经满足的要求

### 1. 一级高约束组已经回到左手核心区

`G2-3` 现在位于 `a/s/d/f` 的 `shift`，已经形成连续块。

这意味着：

- `N19 N20 N21` 三个一级高约束首音重新聚拢。
- 与右手核心乐音区的对位关系重新建立。
- `N24` 作为虚首音，处于同一连续块的最右端，符合当前约束。

### 2. 第二大组三块已经形成完整层次

第二大组当前是三条并排的左手 `shift` 带：

- 上排：`G2-2`
- 中排：`G2-3`
- 下排：`G2-1`

这虽然和最初“优先占左手 base 核心位”的设想不同，但在当前乐音优先方案下，是一种清晰且可记忆的替代结构。

### 3. 第一大组中两块已经成对收拢

`G1-1` 与 `G1-2` 现在分别占据：

- 数字行 `base`
- 数字行 `shift`

形成一组上下对应的双层块：

- `2 3 4 5 / base` -> `G1-1`
- `2 3 4 5 / shift` -> `G1-2`

这有两个优点：

- 两组彼此紧邻。
- 记忆结构非常直接。

## 仍可继续优化的部分

### `G1-3` 还不是最理想的终态

当前 `G1-3` 在左侧外缘 `base` 区：

- `` ` `` -> `N22`
- `1` -> `N09`
- `q` -> `N10`
- `a` -> `N11`

它已经形成了一个左边角簇，但还存在两个不足：

1. 不是单排连续块。
2. 组内顺序更接近“边角聚类”，而不是“线性展开”。

相比前一轮，这一步已经把最外角 `` ` `` 留给了最灵活的虚首音 `N22`，把 `N09/N10/N11` 放到更可达的位置。

不过从优先级看，`G1-3` 本来就属于第一大组收束区，因此在这一步停在左侧外缘是可以接受的。

## 当前判断

如果按首音骨架评价当前 `manual_key_layout.json`：

- 一级优先级：已恢复。
- 第二大组连续性：已恢复。
- 第一大组连续性：大体恢复。
- 第一大组精细顺序：仍可继续优化。

## 下一步建议

如果继续优化首音区，建议只做两类小修，不要再大动：

1. 细调 `G1-3` 的四个位置，让它更像单块而不是边角簇。
2. 评估 `G1-1/G1-2` 用数字行 `base/shift` 的双层结构是否符合长期记忆和实际手感。

在此之前，不建议再动已经稳定的：

- `G2-1`
- `G2-2`
- `G2-3`
- 右手乐音核心区
