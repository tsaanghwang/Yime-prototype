# 乐音布局设计依据（已同步到当前布局真源）

本文件记录当前布局中乐音类 Yinyuan ID 的设计理由和记忆规则。Base 层的
22 个乐音与 Shift 层的 11 个乐音均已同步到
`internal_data/manual_key_layout.json`；首音和候选操作见首音布局文档。

`internal_data/musical_group_template.json` 提供乐音分组和组内高、中、低
语义。本文件解释布局选择，但实现事实以唯一布局真源
`internal_data/manual_key_layout.json` 为准。运行时审计表、crosswalk、KLC
和 Rime 键位表属于派生产物，不得反过来定义布局。

## 范围和容量

键盘共有 47 个可分配的 Base 层物理键。本阶段把其中 22 个分配给乐音：

1. 六个主要口腔乐音组，每组三个，共 18 个；
2. `/n/`、`/ng/` 两组的高音和低音，共 4 个。

剩余 25 个 Base 层键已经在 `internal_data/shouyin_layout_skeleton.md` 中
完成分配：

- 21 个用于实首音（非零首音）；
- 2 个用于高频的 `y`、`w`；
- 1 个用于零首音占位；
- 1 个保留键。

候选选择不占这 47 个 Base 音元键。Space、Enter 和 `Shift+1` 都提交
首选；`Shift+2` 至 `Shift+9` 分别选择第 2 至第 9 候选。候选序号与
Shift 数字键的数字一致，共可直接选择 9 个候选。

全部 33 个乐音类 Yinyuan ID 中，未进入 Base 层的 11 个放入 Shift 层：

- `/-i/` 组 `M19 M20 M21`；
- `/er/` 组 `M22 M23 M24`；
- `/m/` 组 `M25 M26 M27`；
- `/n/` 中音 `M29`；
- `/ng/` 中音 `M32`。

这 11 个成员都属于 Shift 层，物理键位均已确定。

## 学习负担优先

本轮布局以减少记忆规则为首要目标，不以理论上的最短击键路径为首要目标。
用户应能通过少数分区和方向规则推导键位，而不是逐个死记 22 个映射。

Base 层乐音只使用以下规则：

1. `/a/ /o/ /e/（ê）` 三组在左手，`/i/ /u/ /ü/` 三组在右手；
2. 左手组三联从左到右为低、中、高；
3. 右手组三联从左到右为高、中、低；
4. 两手因此统一为食指负责高音、中指负责中音、无名指负责低音；
5. `/n/`、`/ng/` 是音节末尾的鼻音乐音，统一交给两手小指；
6. `/n/` 在左，`/ng/` 在右；高音在 Home 行，低音在 Bottom 行。

鼻尾规则可以记成：

> 鼻尾归小指，n 左 ng 右，高中低下。

这里“高中低下”表示高音在中间的 Home 行，低音在下面的 Bottom 行，
不是把中音放在 Base 层；`/n/`、`/ng/` 的中音仍属于 Shift 层。

## 六组三联的结构

六个主要组形成左右各三行的镜像结构：

```text
左手：从左到右低—中—高       右手：从左到右高—中—低

W E R  /o/                    U I O  /u/
S D F  /a/                    J K L  /i/
X C V  /e/（ê）               M , .  /ü/
```

两手使用同一套“手指—音高”关系：

- 食指：高；
- 中指：中；
- 无名指：低。

组的行位遵循以下记忆结构：

- 左手：`/o/` 在 Top、`/a/` 在 Home、`/e/（ê）` 在 Bottom；
- 右手：`/u/` 在 Top、`/i/` 在 Home、`/ü/` 在 Bottom。

`/i/`、`/u/` 位于相邻两行，`/ü/` 单独位于 Bottom 行；这为音节第2和
第4音元之间的 `i/u` 组合保留了较紧凑的关系。`/a/` 位于左手 Home 行，
与其较高使用频率相称。以上只是布局选择的辅助理由；实际学习时以分区和
方向规则为主。

## 鼻尾四键的结构

`/n/`、`/ng/` 的四个 Base 成员形成键盘两侧的纵列：

```text
左小指                 右小指
A = n高 M28            ; = ng高 M31
Z = n低 M30            / = ng低 M33
```

这四个成员固定出现在音节第4位。把它们交给小指，可以让食指、中指和
无名指区域优先容纳六组三联及后续需要安排的高频实首音。高音使用 Home
键，低音使用 Bottom 键；左右两组的规则完全一致。

├─ 2 个高频 y/w Yinyuan ID（首音骨架已经确定）

若实测后确需修改这 22 个乐音，应先明确要修订的设计规则，再通过布局工作台
或唯一布局真源改动，并将最终理由回写本文件。

## 当前实现状态与后续试验

当前 57 个 Yinyuan ID 已全部进入 Base 或 Shift 层，AltGr 层保留但为空。
本文件与首音布局文档共同解释当前 `canonical_yinyuan_vk_layout_v1`：

1. 确定反引号 `` ` `` 保留键将来的用途；
2. 使用 `scripts\layout_workbench.cmd` 进行可视化交换和词库试打；
3. 保存后由 `python tools/run_locked_layout_pipeline.py` 重建 resolved layout、
   crosswalk、KLC 和其他消费者产物；
4. 根据试打结果决定是否修订布局真源及本设计依据。

不得直接修改 KLC、Rime 键表或运行时 JSON 来形成另一套布局。
