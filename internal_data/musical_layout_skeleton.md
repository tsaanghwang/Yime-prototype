# 乐音排键骨架草案

来源：`internal_data/musical_group_template.json`

目的：先固定乐音 11 组的相邻关系、核心区层级、组三联方向和外扩顺序。此文件是正式排键前的骨架草案，不直接等同于最终键位表。

## 核心原则

1. 同音质三联必须相邻安放，不能被其他组打断。
2. 同类口形或同类收束功能的组，应形成连续组块。
3. 先定组块，再定组三联的具体键位。
4. 如果整体方案保持“首音偏左手、乐音偏右手”，则乐音核心组应优先放在右手主键区。
5. 组三联内部顺序固定，但左右手方向应镜像安排。

补充约束：

- 右手区从左向右排列为“高、中、低”。
- 左手区从右向左排列为“高、中、低”。
- 这样两手食指都可以落在各组三联的高音位上。

## 一级分区

建议先把 11 组分成 4 个大区，而不是直接逐组散排：

1. 高元音核心区：`/i/` `/ü/` `/u/`
2. 开口元音核心区：`/e/（ê）` `/o/` `/a/`
3. 特殊韵质区：`/-i/` `/er/`
4. 鼻尾收束区：`/m/` `/n/` `/ng/`

理由：

- `i/ü/u` 在口形和高位性质上更接近，适合作为一个连续核心块。
- `e/o/a` 可作为另一块开口度更大的连续区。
- `-i` 与 `er` 都更像特殊化韵质，宜单列为过渡区。
- `m/n/ng` 都具有收束、尾化特征，宜并列成一条收尾带。

## 建议宏观布局

如果采用“首音左手，乐音右手”的总框架，乐音骨架建议如下：

```text
右手上层:   /i/    /ü/    /u/
右手中层:   /e/    /o/    /a/
右手内侧:   /-i/   /er/
右手外侧:   /m/    /n/    /ng/
```

这不是逐键图，只是组块图。

含义：

- 右手上层放高元音核心区。
- 右手中层放开口元音核心区。
- `/-i/`、`/er/` 放在核心区和鼻尾区之间，作为过渡带。
- `/m/`、`/n/`、`/ng/` 放在最外侧，形成稳定的尾化收束区。

这一步先优先处理六个核心组：`/i/ /ü/ /u/ /e/ /o/ /a/`。

2. `/o/`：864
3. `/u/`：856
4. `/i/`：648
5. `/e/`：586
6. `/ü/`：273

因此，第三排（`f`、`j` 所在排）应优先留给更高频的核心组。

## 六个核心组的第一版骨架

先不碰其余 5 组，只给六个核心组定一个可落地的第一版块位：

```text
左手第二排:  /o/   -> W E R
左手第三排:  /a/   -> S D F
左手第四排:  /e/   -> X C V

右手第二排:  /i/   -> U I O
右手第三排:  /u/   -> J K L
右手第四排:  /ü/   -> N M ,
```

说明：

- 左手三联统一按“右到左：高、中、低”读取。
- 右手三联统一按“左到右：高、中、低”读取。
- 左手核心第三排给 `/a/`，右手核心第三排给 `/u/`，因为它们分别是各自三组里最高频的组。
- `/o/` 与 `/i/` 次一级，先放第二排。
- `/e/` 与 `/ü/` 频次更低，先放第四排。

## 五个外围组的层级策略

对 `/-i/`、`/er/`、`/m/`、`/n/`、`/ng/` 五组，不宜再套用“所有成员都占 base 连续三键”的规则，而应按实际用途分层处理。

按 `yinjie_code.json` 中的实际出现次数，这五组目前为：

1. `/n/`：448，其中 `M28=168` `M29=80` `M30=200`
2. `/ng/`：375，其中 `M31=140` `M32=71` `M33=164`
3. `/-i/`：90，其中 `M19=31` `M20=28` `M21=31`
4. `/m/`：30，其中 `M25=10` `M26=10` `M27=10`
5. `/er/`：15，其中 `M22=5` `M23=5` `M24=5`

结合直接后继首音约束，可得到下面的排键原则。

### `/-i/` 组

- 只与齿音和齿龈-硬腭间音相关首音直接相拼，范围很窄：`N12 N13 N14 N15 N16 N17 N18`。
- 频次不高，但它确实参与首音直接拼合，因此不宜整组压到 `shift` 层。
- 结论：放在 base 层边缘区最合适，既不要抢核心位，也不要沉到 `shift` 层伤害直接拼合效率。

### `/er/` 组

- 实际总频次最低。
- 直接后继只见于 `N22`，也就是零首音占位；从排键角度看，等价于“不需要围绕高优先首音做核心对位”。
- 结论：整组放 `shift` 层最合理。

### `/m/` 组

- 总频次很低。
- 直接后继只见于 `N11`、`N22`，也不应牵动核心首音布局。
- 结论：整组放 `shift` 层是合理的，优先级和 `/er/` 同类。

### `/n/` 组

- 总频次明显高，且高音 `M28` 与低音 `M30` 远高于中音 `M29`。
- 它主要承担尾音功能，频次高，但并不要求把高优先首音组一起拖到边缘。
- 结论：`M28`、`M30` 应保留在 base 层外圈可达位；`M29` 可上移到 `shift` 层。

### `/ng/` 组

- 情况与 `/n/` 类似：高音 `M31` 与低音 `M33` 高频，中音 `M32` 明显偏低。
- 它同样主要是尾音负荷，而不是首音直接拼合负荷。
- 结论：`M31`、`M33` 保留在 base 层外圈可达位；`M32` 可上移到 `shift` 层。

## 五个外围组对首音布局的影响

这里最关键的一点是：

- `/n/`、`/ng/` 高频，说明它们要有顺手 base 位。
- 但它们的直接后继首音关系很弱，只集中在 `N11`、`N22` 这样的非一级核心约束首音上。
- 所以不应该为了 `/n/`、`/ng/` 去把一级、二级首音组整体往边缘推。

换句话说，正确做法不是“让首音跟着 `/n/`、`/ng/` 走”，而是：

1. 先保住高优先首音和六个核心元音组的中心关系。
2. 再在右手外圈给 `/n/`、`/ng/` 的高低音留出 base 位。
3. 把它们的中音成员挪到 `shift` 层，减少对核心区的挤占。

## 五个外围组的建议落层

第一版建议如下：

1. `/-i/`：base 层边缘连续三键
2. `/er/`：shift 层整组三键
3. `/m/`：shift 层整组三键
4. `/n/`：高音、低音在 base 层；中音在 shift 层
5. `/ng/`：高音、低音在 base 层；中音在 shift 层

这意味着外围组允许出现两种模式：

1. 整组三键同层
2. 高频成员留 base、低频成员上 shift 的拆层模式

## 五个外围组的第一版候选键位

在不动六个核心组的前提下，右手外圈可先采用下面这一版候选键位：

```text
/-i/  :   P   [   ]
/er/  : Shift+P  Shift+[  Shift+]

/n/   :   ;   Shift+;   '
/ng/  :   .   Shift+.   /

/m/   : Shift+0  Shift+-  Shift+=
```

对应含义如下：

- `/-i/`：`P=高` `[` `=中` `]=低`
- `/er/`：`Shift+P=高` `Shift+[=中` `Shift+]=低`
- `/n/`：`;=高` `Shift+;=中` `' =低`
- `/ng/`：`.=高` `Shift+.=中` `/=低`
- `/m/`：`Shift+0=高` `Shift+-=中` `Shift+=低`

这样安排的理由是：

1. `/-i/` 放在 `P [ ]`，仍然属于右手外缘 base 区，但没有沉到 `shift`，可以保住它和 `N12-N18` 直拼时的效率。
2. `/er/` 与 `/-i/` 共享同一物理键簇的 `shift` 层，记忆关系最直接。
3. `/n/` 用 `;` 和 `'` 保留高低两个 base 位，而且高音落在更靠食指的 `;` 上。
4. `/ng/` 用 `.` 和 `/` 保留高低两个 base 位，而且高音落在更靠食指的 `.` 上。
5. `/n/` 与 `/ng/` 的中音分别放到 `Shift+;`、`Shift+.`，既保留组内对应关系，又不挤占核心 base 位。
6. `/m/` 频次最低的一档之一，且不需要围绕高优先首音布局，因此可以放到更远的右上外缘 `shift` 位。

## 这版候选键位对首音区的含义

如果采用这版候选键位，首音区可以继续保持原来的优先级，不需要为鼻尾组让位：

1. 一级首音组仍优先围绕 `/i/ /ü/ /u/ /a/ /o/ /e/` 六个核心组布局。
2. `/-i/` 只占右手最外缘的一小条 base 带，不会反推核心首音外移。
3. `/n/`、`/ng/` 虽然高频，但只占右手外缘的 4 个 base 键和 2 个 `shift` 键，也不会压缩首音核心位。
4. `/er/`、`/m/` 退到 `shift` 层后，首音和核心乐音之间的主通道仍然干净。

## 六个核心组的组三联方向

按上面的第一版块位，组三联内部顺序应固定为：

1. 左手：右到左为“高、中、低”
2. 右手：左到右为“高、中、低”

因此具体写成：

- `/a/` 组：`F=高` `D=中` `S=低`
- `/o/` 组：`R=高` `E=中` `W=低`
- `/e/` 组：`V=高` `C=中` `X=低`
- `/u/` 组：`J=高` `K=中` `L=低`
- `/i/` 组：`U=高` `I=中` `O=低`
- `/ü/` 组：`N=高` `M=中` `,=低`

## 组间相邻要求

- `/i/` 必须紧邻 `/ü/`
- `/ü/` 必须紧邻 `/u/`
- `/e/（ê）` 必须紧邻 `/o/`
- `/o/` 必须紧邻 `/a/`
- `/-i/` 应尽量紧邻 `/e/（ê）` 与 `/er/`
- `/er/` 应尽量紧邻 `/-i/` 与鼻尾区
- `/m/`、`/n/`、`/ng/` 三组必须连续展开

## 建议优先顺序

正式排键时，建议先固定以下四个层次：

1. 先固定六个核心元音组：`/i/ /ü/ /u/ /e/ /o/ /a/`
2. 再插入特殊韵质组：`/-i/ /er/`
3. 最后放置鼻尾收束组：`/m/ /n/ /ng/`
4. 再回头细调各组三联内部的具体落点

理由：

- 六个核心元音组承担的组块地位最高，先定它们，整体骨架才不会散。
- `/-i/`、`/er/` 数量少但性质特殊，更适合作为连接层，而不是主骨架起点。
- 鼻尾组更像收束带，布局上宜服从前面两层，而不是反过来牵动核心区。

## 组三联内部顺序

核心组六个成员应保持统一方向，不要某组横排、某组竖排、某组反向。外围五组则允许在保持“高音朝食指”原则下做拆层处理。

建议统一为：

1. 左手从右到左：高调（高平）、中调（半高平）、低调（低平）
2. 右手从左到右：高调（高平）、中调（半高平）、低调（低平）

也就是：

- 如果三联在左手横排，顺序为“右高、中中、左低”。
- 如果三联在右手横排，顺序为“左高、中中、右低”。
- 如果后续某组三联改成竖排，也应保持“靠食指的一端是高音位”的原则。
- 如果外围组需要拆层，也应优先保证高音位和高频低音位留在 base 层。

这样做的好处是：

- 11 组三联的记忆法完全一致。
- 一眼能看出高、中、低的方向，不会组间反转。
- 后续若要把音高变化映射为手指移动方向，也更容易统一。

## 建议占位区

以下仍然是组块级建议，不是最终逐键定稿：

1. 一级核心区：右手主键中上区
适合：`/i/` `/ü/` `/u/` `/e/` `/o/` `/a/`
目前第一版骨架优先使用 `w e r`、`s d f`、`x c v`、`u i o`、`j k l`、`n m ,` 这六个连续区域。

2. 二级过渡区：右手内侧靠中缝区域
适合：`/-i/` `/er/`
建议与核心元音区保持一步可达，不要被鼻尾组隔开。

3. 外围收束区：右手外缘或下外侧
适合：`/m/` `/n/` `/ng/`
建议保持三组连续，方便统一记忆为“鼻尾带”。

## 暂不锁死的部分

以下内容在这一步暂不锁死：

1. `/-i/` 与 `/er/` 谁更靠近核心区。
2. `/m/ /n/ /ng/` 三组在收束区内部的精确顺序。
3. 六个核心组最终是否要整体左移或右移一列。
4. `/o/` 与 `/i/` 是否需要互换到更贴近中缝的位置。

这些需要后续再叠加两类信息：

1. 实际使用频次和连拼频次。
2. 与高约束首音组之间的跨手路径。
