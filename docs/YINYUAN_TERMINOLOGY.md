# 音元系统术语说明

## 文档定位

本文档用于说明音元系统中若干核心术语的中文含义、英文译法与相互关系。

**总索引与 AI 提醒**（含速查表、干音/乐音误解、代码命名）：请先读 [TERMINOLOGY_INDEX.md](TERMINOLOGY_INDEX.md)。
**代码类名唯一约定**：[syllable/NAMING.md](../syllable/NAMING.md)。

本文档不试图把音元系统硬套进传统音位学或传统语音学的既有术语，而是说明：

1. 哪些术语可以借用已有英文词汇。
2. 哪些术语必须保留音元系统自己的定义。
3. 这些术语与传统音位、音素、音段、声调层等分析框架的差异在哪里。

## 核心判断

从更一般的角度说，音元系统不是只为声调语言建立的局部分析法，而是一个允许根据语言中的区别性语音属性来切分片音、归并音元的更一般框架。

这意味着：

1. 音元系统不预先把“只按音质分析”当作唯一合法方式。
2. 它也不预先把音高、音强或音长都降为整个音节的附属特征。
3. 不同语言可以在这个框架内呈现不同特例。
4. 在现代通用汉语这一特例中，音高和音质是主要的区别性特征。

因此，音元系统中的基本单位，不能直接等同于传统意义上的 phoneme、phone 或 segment；而在现代通用汉语的讨论里，它又确实表现为一种主要按音高和音质来组织的单位。

## 术语总表

### 1. 时段

- 中文：时段
- 英文：temporal slot

说明：

这里的 slot 不是纯抽象占位符，而是语流中一个可被语音单位占据的时间位置或时间段。

如果需要更简短，也可以在上下文清楚时直接写 slot，但正式定义时建议写 temporal slot。

### 2. 片音

- 中文：片音
- 英文：phonic slice

说明：

片音不是传统语音学里现成的 phone，也不宜直接译为 phonetic segment。

原因在于：

1. 片音是从时域亦即音值上切分出来的音片（语音片段）。一般地说，在切分片音时，同时以音高、音强、音质和音长四个属性（特征）为确定片音的依据。然而，在切分音素或音段时，通常主要以音质为依据，其他属性或不予考虑或被降为附属特征或设置成默认值。
2. 它有时小于传统音素或音段，例如把一个较长元音或鼻音按时域切成前后两段或三段。
3. 它有时又可以与传统的音素或音段重合，例如某些首辅音。

因此：

- 片音不能稳定地定义为 phonetic segment。
- 片音更适合定义为 phonic slice，即从时域亦即音值上切分出来的音片（语音片段）。

### 3. 音元

- 中文：音元
- 英文：yinyuan

说明：

音元是音元系统中的抽象单位。它的定义包含三层要点：

1. 它是一个占据时段的单位。
2. 它是按某语言中实际起区别作用的语音属性来划分出来的单位。
3. 它不是单独对应某一个片音，而是由一类片音来实现。在音节结构、拼音表示或程序运行时，一个音元可以理解为一个变量，片音则是这个变量在某个时刻的取值。

因此，音元不宜简单译为：

- phoneme
- segment
- slot-based phonological unit

前两者会把它拉回传统音位或音段框架；最后一个虽然表达了“时段单位”，但省略了“由语言内部的区别性属性来划分”这一层核心含义。

在英文中，目前最稳妥的做法是把它直接写作 `yinyuan`，并在需要时补一句解释性的定义。

如果语境明确限定在现代通用汉语，也可以把它临时解释为一种 `pitch-quality slot unit`。但这种说法应理解为汉语特例下的工作性释义，而不宜当作整个理论框架的总定义。

把 `yinyuan` 作为主标签的好处是：

1. 可以避免把汉语特例误写成总理论。
2. 可以保留该术语在更一般框架下继续扩展的空间。
3. 可以在不同语言讨论中，按需要补入不同的解释性限定。

### 4. 片音与音元的关系

片音与音元的关系，不是“一一对应”的映射关系，而更接近：

- 音素与音位
- 实例与抽象类
- 程序运行时的取值与变量

可以这样理解：

1. 片音是从时域亦即音值上切分出来的音片（语音片段）。
2. 音元是把这些片音按某语言中的区别性语音属性归并后的抽象单位。
3. 一个音元可以被一组片音实现，而不是只对应某一个片音。

因此，“音元对应片音”这个说法不准确；更准确的说法是：

- 音元由一类片音实现。
- 片音是音元的具体实现值。

## 推荐定义句

### 中文定义句

可采用如下定义：

> 音元是音元系统中的抽象时段单位，是按某语言中实际起区别作用的语音属性来划分、并占据一定时段的单位；它不是单独对应某一个片音，而是由一类片音来实现。在音节结构、拼音表示或程序运行时，一个音元可以理解为一个变量，片音则是这个变量在某个时刻的取值。

### 对片音的中文定义句

> 片音是从时域亦即音值上切分出来的音片（语音片段）；它有时小于传统音素或音段，有时又与传统音素或音段重合，因此不能简单等同于传统语音学中的音素或音段。

## 推荐英文定义句

### 音元

> A yinyuan is an abstract unit occupying a temporal slot in the speech stream and defined by the contrastive speech properties relevant in a given language. It is not mapped to any single phonic slice, but is instantiated by a class of phonic slices.

### 片音

> A pianyin is a phonic slice, namely a speech-sound slice cut in the temporal dimension. It may be smaller than a traditional segment, though in some cases it may coincide with one.

## 片音的人机衔接意义

从理论落地的角度看，片音还可以被理解为连接可感知语音事实与可计算语音表示的中介单位。

这意味着：

1. 片音不是单纯的工程帧。
2. 片音也不是完全脱离声学实现的纯抽象实体。
3. 在工程实现上，一个片音通常由一个或若干连续声学帧所承载。
4. 帧提供短时声学观测，片音构成对帧序列的结构化切分结果，音元则是对片音的进一步抽象归并。

如果需要从语音技术角度理解片音，可以把它表述为：人能发出、能听辨，同时机器也能稳定切分、表征和重建的最小语音片段。

这层说明的详细展开，见：[片音与语音技术单位的对应关系](PIANYIN_TECH_BRIDGE.md)。

## 从一般框架到语言特例

从更一般的角度说，音元系统允许根据多种可区别的语音属性来切分片音并确定音元。在较宽的理论设想中，这些属性可以包括：

1. 音高
2. 音强
3. 音质
4. 音长

不过，不同语言真正用来区别语义的属性并不相同。因此，不同语言中的音元系统，可以看作这个更一般框架的不同特例。

### 只以音质区别语义的语言

如果一个语言只用音质来区别语义，而音高、音强和音长都不承担区别功能，那么它的音元系统实际上就退化为音位系统。

在这种情况下：

1. 音元仍可作为理论名称保留。
2. 但分析结果与传统音位分析没有实质差别。

一个假设性的简单例子是：某语言只有 `pa`、`ba`、`ta`、`da` 这类对立，所有音节都具有可预测的默认音高、默认音强和默认音长，因此真正承担语义区别的只有辅音或元音的音质差异。对于这类语言，音元系统就等价于音位系统。

如果忽略英语中的若干复杂问题，例如重音、时长和语调带来的影响，那么英语也可以近似看作这一类型的例子。

### 同时以四属性区别语义的语言

如果一个语言同时用音高、音强、音质和音长来区别语义，那么它的音元系统就表现为完备意义上的全属性音元系统。

在这种语言中：

1. 四种属性都不是附属的或默认的。
2. 四种属性都可能成为切分片音和归并音元的区别性依据。

一个假设性的例子是：某语言中，`a`、高音的 `á`、强读的 `a`、长音的 `aː` 都分别构成不同语素，甚至某些形式还可能因“高音 + 强读 + 长音”的不同组合而继续形成新的语义对立。对于这样的语言，音元系统就不再退化为传统音位系统，而是呈现为理论上最充分展开的音元系统。

就目前所知，现实中还没有一个已被充分证明的语言，恰好把音高、音强、音质和音长四种属性都系统地用作同等地位的区别性特征。因此，这种情形目前更适合作为理论上的完备模型来理解。

### 现代通用汉语的位置

现代通用汉语可以看作上述更一般框架中的一个特例。

在现代通用汉语中，一般地说：

1. 音高和音质是主要的区别性特征。
2. 音强和音长通常不直接承担语素意义的区别功能。

因此，在讨论现代通用汉语的音元系统时，可以先把它表述为一种主要根据音高和音质来切分片音并据此确定音元的分析方法；而音强和音长，则保留在更一般的理论框架中，作为适用于其他语言或更细分析场景的潜在区别性维度。

在声调语言中，音元系统与传统音位系统的差异尤其明显。

传统音位系统通常：

1. 主要按音质切分语音。
2. 把声调看成整个音节的附属特征，或另行处理。

音元系统则：

1. 在语言需要时，直接按音高与音质等区别性属性共同切分语音。
2. 在形式上把辅音类噪音的音高也纳入系统，只是其音高值记为零或空。
3. 不把声调仅当作整个音节的附属特征。
4. 也不把声调单独外置成独立 tier。

但在只由音质承担语义区别的非声调语言中，音元系统可以退化为传统音位系统。也就是说：

1. 在这类语言里，音元与音位分析的结果可以等价。
2. 两者不是互相对立的两套系统，而是一套更一般的分析系统与它的特例之间的关系。

## 音节结构术语（实现链常用）

下列术语用于 **Yinjie 四元模型** 与 `syllable/` 编解码链，与上一节的「片音 / 音元」处于不同但相关的层次：**结构段** vs **片音/音元类别**。

这里的 canonical 等长四字符结构，推荐称为 **四元模型**；需要强调结构等长时，可写 **等长四元模型** 或 **四元音节模型**。早期「四音元位模型」「四槽模型」不再作为主术语使用。

如果需要描述 `yinjie` 四字符编码，请再分出第三层：**音元位 / 编码位**。四字符中的第 1 至第 4 位，是固定顺序上的 **位置**；这些位置里依次存放的，是 **首音元 / 呼音元 / 主音元 / 末音元**。因此：

1. 可以说 **四元模型**、**四元编码**，或在强调位置时说 **第 1 至第 4 音元位**。
2. 可以说 **首音元 / 呼音元 / 主音元 / 末音元**。
3. **不要**把“四音元位”当作模型主名反复使用，也不要把“音元位”简写成“音位”。在汉语语音学语境里，“音位”默认对应 `phoneme`，容易造成误读。

| 中文   | 推荐标识   | 说明                                                                                                                                       |
| ------ | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 音节   | yinjie     | 分析单位；分为首音 + 干音                                                                                                                  |
| 首音   | shouyin    | **声母及与其联结的调段**构成的音段（通俗：**声母 / 辅音**）；真源见 `syllable/analysis/syllable.py`；**就具体首音而言，实际就是声母**      |
| 干音   | ganyin     | **韵母及与其联结的调段**构成的音段（通俗：**带声调的韵母**）；真源见 `syllable/analysis/syllable.py` 的 `Ganyin`；**不是**「乐音」的同义词 |
| 乐音   | yueyin     | 有稳定调段的片音/音元 **类别**；填充干音内呼音、主音、末音（峰前/峰/峰后段）                                                               |
| 噪音   | zaoyin     | 首音侧片音/音元类别                                                                                                                        |

| 末音   | —          | **Post-peak** segment (`descender`); **not** rime coda                                                                                     |

结构关系（与 `syllable/codec/yinjie.py` 一致；**禁止改层级**，详见 [TERMINOLOGY_INDEX.md](TERMINOLOGY_INDEX.md)「音节结构」节）：

**结构段** 与 **音元类别** 是两条轴：首音是「声母 + 联结调段」、干音是「韵母 + 联结调段」这两类 **音段**；噪音 / 乐音是 **类别**，分别填充首音段与干音内子段，不与首音/干音同级。

```text
音节
├── 首音（声母 + 联结调段；声母/辅音）  ← 材料：噪音类音元
└── 干音（韵母 + 联结调段；带调韵母）  ← ≠ 乐音；材料：乐音类音元（呼/主/末）
    ├── 呼音（峰前段）                 ← ≠ 韵头/介音
    └── 韵音
        ├── 主音（峰段）
        └── 末音（峰后段）             ← ≠ 韵尾/coda
```

音段层严格定义见 [`syllable/analysis/syllable.py`](../syllable/analysis/syllable.py)；Yinjie 内部分解见 [`syllable/codec/yinjie.py`](../syllable/codec/yinjie.py)。

### 常见误解

1. **首音 ≠ 噪音。** 首音是 **声母及与其联结调段** 构成的音段（通俗即声母/辅音）；噪音是 **类别名**，指填充首音段的片音/音元。**就具体首音而言**，「首音」与「声母」指同一音段。
2. **干音 ≠ 乐音。** 干音是 **韵母及与其联结调段** 构成的音段（带调韵母）；乐音是 **类别名**，指有稳定调段的片音/音元。一个干音音段在 Yinjie 四元模型内 **由多段乐音（及韵音划分）填充**。
3. **干音 ≠ 无调韵母。** 编码与数据面以 `final + tone` 为键（如 `i1`），见 `GanyinEncoder` 与 `yinyuan/ganyin.json`。
4. **`ganyin` 与 `yueyin` 不可互换。** 代码中 `GanyinEncoder` 处理干音 **音段/结构**；`YueyinYinyuan` 命名的是乐音 **音元** 类，不表示「干音的另一个名字」。
5. **呼音 ≠ 韵头 / 介音；末音 ≠ 韵尾 / coda。** 呼音、主音、末音是 Yinjie 内 **峰前 / 峰 / 峰后** 结构段（`yinjie.py`）；其 **填充** 均为「某音质段 + 与其联结的调段」构成的乐音 **音段**（见 sound_variable_analysis 流程）。例如：三质/后长呼音 = 由韵头及与其联结的调段构成的音段；前长呼音 = 由韵腹前段及与其联结的调段构成的音段；单质呼音 = 由韵母前段及与其联结的调段构成的音段。韵头、韵尾等是 **音质切分名**，不可与结构段互换。
6. 外部二手资料若将「首音/噪音/声母」或「干音/乐音/韵母」或「呼音/韵头」「末音/韵尾」混为一谈，**以本仓库结构定义与 [TERMINOLOGY_INDEX.md](TERMINOLOGY_INDEX.md) 为准**。
7. **音元位 ≠ 音位。** `Yinjie` 四字符里的“位”是编码顺序位置；“首音元 / 呼音元 / 主音元 / 末音元”是填入这些位置的音元。模型整体称 **四元模型**，位置说明才称 **音元位 / 编码位**；不要把位置名缩成“音位”，也不要把它译回 `phoneme`。

代码层唯一类名见 [syllable/NAMING.md](../syllable/NAMING.md)。

## 推荐使用方式

在中文正文中，建议保留：

- 时段
- 片音
- 音元

在英文正文中，建议第一次出现时写为：

- temporal slot
- pianyin (phonic slice)
- yinyuan

如果语境明确是在讨论现代通用汉语，也可以在第一次出现时写为：

- yinyuan (a pitch-quality slot unit in the Modern Standard Chinese case)

后续如果上下文清楚，可以直接使用：

- slot
- pianyin
- yinyuan

这样既能保持术语稳定，又能避免把音元系统误写成传统音位学术语的简单别名。
