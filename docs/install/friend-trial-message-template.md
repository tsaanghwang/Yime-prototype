# 发给朋友的说明模板

这份模板只服务一个目标：

- 你把安装包发给朋友时，顺手附上一段足够短、但能把预期说清楚的话。

## 最短可直接发版

可以直接复制下面这段，再按你这次版本号和重点微调：

```text
我这边做了一个 Windows 输入法原型的初级版，想请你帮我试装一下。

安装包：Yime-Setup-0.1.0.exe

这还不是系统级正式输入法产品，而是一个可安装的原型应用。现在最想确认的是：
1. 能不能顺利安装
2. 安装后能不能正常启动
3. hotkey 能不能把候选框叫出来
4. 能不能正常上屏

如果你愿意试，最好在记事本这种最简单的输入框里先测一下。

如果出问题，麻烦尽量告诉我是卡在下面哪一步：
- 安装时报错
- 安装后打不开
- 能打开但 hotkey 叫不出来
- 候选框能出来但不能上屏
- 第一次正常，重开后失效

你不用长篇写反馈，几句话把现象描述清楚就够了。
```

## 更自然一点的版本

如果你不想发得太像测试说明，可以用这一版：

```text
我最近做了个 Windows 输入法原型，先做了一个能安装的版本。你要是方便的话，帮我试装一下？

我现在主要想确认它在别人电脑上是不是也能顺利完成这几步：安装、启动、唤起候选框、正常上屏。

你可以先在记事本里简单试一下；如果哪里不对，告诉我大概卡在安装、启动、唤起还是上屏这一步就行。
```

## 建议你自己先替换的 3 个地方

1. 把安装包文件名改成你这次实际发送的版本。
2. 如果这次你特别想测某一项，例如 hotkey 或用户词库，就在正文里明确点出来。
3. 如果你知道对方不熟悉命令行或输入法原型，就不要塞太多技术解释，保持短一点。

## 不建议一起发太多的东西

- 不要一上来就贴完整开发背景
- 不要把所有已知限制一次讲完
- 不要附过长的测试步骤

第一次发给朋友试装，目标不是“把所有事情解释完”，而是先拿到一轮有效反馈。
