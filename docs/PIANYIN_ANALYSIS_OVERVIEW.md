# 片音分析与音元表示：工程阅读概要

## 文档地位

本文是 AI 和开发者阅读 `syllable/analysis/`、片音对象、音元对象及 Yinyuan ID 数据时的工程解释入口。它说明代码中的对象代表什么、当前程序实际做了什么，以及哪些语音实现仍只是待验证设想。

如果旧理论稿、历史注释或类名使人误以为片音是等长平顶音段、固定时间窗、原始波形块或已经由程序检测出来的声学单位，以本文的工程解释为准。本文不修改音节编码规则，也不建立新的拼音到 Yinyuan ID 映射。

## 先记住四个层次

```text
连续语音中的特征样本
  -> 用不等长特征段表示为片音（pianyin）
  -> 将片音按规则归并为抽象音元（yinyuan）
  -> 用 Yinyuan ID（M01、N01 等）唯一标识音元
```

这里的“样本”不是数字信号处理中按固定采样率取得的瞬时幅值，也不是按等长时间窗截取的波形。它指从连续语音中观察或抽取的、能够代表某种音高和音质状态的特征样本。

片音是表示这种样本的特征段。片音时长可以不同，内部和边界可以存在连续的音高变化、音质过渡和协同发音。片音具有某个单一音高特征和音质特征，是说它可以被发出并被听知为一个质调类别，不是说整个音段的基频、共振峰或其它声学参数保持恒定。

音元是表示片音的变元。一个音元可以根据分布环境取用不同的条件片音；片音是音元在特定环境中所取的音值。Yinyuan ID 只是音元的稳定唯一编号，不是音标、录音、波形、键位或实际音值。

## 代码阅读时禁止作出的推断

1. 看见 `M01`，不能直接断言它就是一段可独立剪取的平顶 `[i⁵]`。ID 标识的是抽象音元；音标式写法至多是特征说明或某种条件片音的记法。
2. 看见四个 Yinyuan ID，不能推断音节被四个等长时间窗平均切开。四元模型首先是固定的分析和编码位置；相应片音在语音实现中的时长可以不同。
3. 看见 `Pianyin`、`ZaoyinPianyin` 或 `YueyinPianyin` 对象，不能把它们理解为音频缓冲区、声学帧或程序已经从波形中检测出的音段。当前对象保存的是符号化分析结果。
4. 看见 `tone`、调号或五度值，不能推断片音内部存在恒定不变的平顶基频。它们表示分析所取的音高类别、目标或特征状态。
5. 看见片音序列，不能假定把几段录音机械首尾连接就能恢复音节。语音实现原则上还需要时长配置、插值、边界平滑和协同发音处理。
6. 当前仓库能生成音元编码，不等于片音的自然边界、最短听知时限及普遍心理现实已经得到实验验证。

## 当前程序实际实现的链

当前生产链从带调拼音字典开始，不从语音波形开始：

```text
带调拼音
  -> SyllableEncodingPipeline 切分首音段和干音段
  -> ShouyinEncoder / GanyinEncoder
  -> 四个 Yinyuan ID
  -> 三模式编码和布局投影
```

因此，`syllable/analysis/` 目前实现的是标准拼音到符号化片音/音元结构的规则分析和编码，不是自动语音识别中的波形切分器，也不是文本到语音的声学合成器。

主要代码角色如下：

- `syllable/analysis/syllable_encoding_pipeline.py`：规范拼音的正式分析入口。
- `syllable/analysis/shouyin_encoder.py`、`ganyin_encoder.py`：把首音和干音分析结果转换成 Yinyuan ID。
- `syllable/analysis/pitched_pianyin.py`：乐音类片音的符号化表示，不保存波形。
- `syllable/analysis/yueyin_mapper.py`：片音特征到乐音音元的归并规则。
- `syllable/analysis/yueyin_yinyuan.py`、`zaoyin_yinyuan.py`：音元对象及片音到音元的转换边界。
- `syllable/yinyuan/*_enhanced.json`：音元语义数据；其中的 `M01`、`N01` 等是 Yinyuan ID。

## “能单发、能听知”的工程含义

“能单发”表示发音者能够有意产生近似某个片音质调特征的短音；“能听知”表示听者能够把这个短音辨认为相应的质调类别。这两个说法不要求片音等长、平顶或边界绝对清楚，也不证明自然语流中存在可以不经处理直接剪出的独立声块。

如果以后增加语音合成试验，较合理的数据流是：

```text
音元序列
  -> 根据上下文为音元选择条件片音及质调目标
  -> 分配不等时长
  -> 在相邻目标之间插值、平滑并处理协同发音
  -> 生成连续音节
```

插值和重建目前是说明片音序列怎样可能实现为连续语音的方向，不是本仓库已经完成的功能。

## 已成立与待验证的边界

已经由工程实现验证的是：对于现行有来源的规范带调音节，正式规则链能够生成稳定的四个 Yinyuan ID，并用于三模式编码、布局投影和词库生成。

仍待语音实验验证的是：片音类别能否被稳定产生和听知、片音边界怎样测定、最短听知时限是多少、不同说话人和语速下是否保持一致，以及根据片音目标进行插值能否充分重建自然音节。

因此，读代码时应采用下面的结论：

> 片音分析和音元分析目前是已经落地的符号化音节分析与输入编码模型；片音作为自然语音最小单位的语音学地位，仍是可检验而尚未完成实验验证的理论主张。
