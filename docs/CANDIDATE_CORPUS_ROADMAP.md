# 候选语料库整理路线图

## 文档定位

本文记录统一候选语料库在完成来源合并、拼音门禁、轻声范围约束和运行库重建之后的工程现状，并规定
下一阶段的整理顺序。它是“现在整理到哪里、下一步整理什么”的长期入口。

这里使用“候选语料库”，而不简单称为“词库”，因为全集同时包含词、专名、术语、固定表达、能产
组合、句法片段、语料切片和噪声。系统不预先假定每个高频字串都是词。

相关文档分工如下：

- [字词拼音与频次统一语料包](LEXICON_SOURCE_BUNDLE.md)：来源、构建口径和产物格式；
- [汉语输入候选整理与动态组合系统](INPUT_CANDIDATE_MODEL.md)：分类、决策覆盖层和动态组合器；
- 本文：现状快照、质量判断、整理优先级和分阶段路线；
- [当前实现总览](CURRENT_ARCHITECTURE.md)：整个拼音—音元—运行候选生产链的工程事实入口。

## 三层必须分开

```text
统一来源语料库 source_lexicon.sqlite3
  保存完整来源事实、读音、频次、分类和拒绝证据
              │
              ▼
候选整理覆盖层 input_model.sqlite3
  保存 proposed / approved / rejected / deferred 决策
              │
              ▼
运行候选 pinyin_hanzi.db
  只承载当前发布所需的静态候选和编码结果
```

来源语料库应尽量完整并可追溯；整理覆盖层可以逐步积累稀疏判断；运行候选以后可以缩减。不得为了
缩小运行词库而删除来源证据，也不得把尚未审查的机器建议直接写成生产事实。

重新构建后的 `manifest.json` 为准，不把本节数字当作永久锁定值。

### BCC 覆盖

| 项目 | 数量 |
| --- | ---: |
| BCC 不同字串 | 2,101,047 |
| 已匹配合规读音 | 433,971 |

未解码数量很大，不等于存在 166 万个真正的词尚未注音。BCC 字串中包含偶然相邻组合、句子截片、

性质，而不是给所有字串逐字猜读音。

BCC 单字已经基本覆盖：20,261 个不同单字中有 20,244 个具备合规读音，只余 17 个。真正的整理压力
已经从单字注音转向多字组合判断。

### 多读音与轻声

- 22,989 个不同字词存在多个合规读音，共形成 50,054 条冲突输出记录；
- 这些冲突可能是真多音、轻声、本调、文白异读、专名读音或来源错误，不能一律删成单音；
- 当前有 14 条读音被限制为 `word_context_only`，只保存词境证据，不生成孤立单字候选；
- `藍` 已验证这一机制：保留单字 `lan2`，词境 `lan5` 不生成单字候选，过时的 `la5` 被定点排除；
- 轻声由完整来源词音和来源标调制度约束，不再用早期“五声补齐”生成。

## 已经完成的整理能力

### 1. 来源边界已经闭合

- 只读取真实本地上游文件；
- BCC 各分域原始计数分别保存，汇总频次取各分域最大值；
- 不把二次合并频次文件重新当作来源；
- 万象的 `cuoyin`、`mixed` 默认不进入拼音解码来源；
- 每个输入文件在 manifest 中保存路径、摘要和大小。

### 2. 拼音进入编码前有统一门禁

- 非法字符、异常拼式和缺失注音可以拒绝并报告；
- 新音节必须有真实来源和明确审查，不自动补齐声调；
- 不猜读音，不手写音元码、Yinyuan ID 或键位；
- 过时读音可以定点排除，不扩张成全局音节禁用。

### 3. 轻声与词境读音已经分层

- 记录轻声位置和来源解释状态；
- 区分完整候选读音与仅供词境使用的证据；
- 单字本调、词汇性轻声和临时语流弱化不再压成同一张五声闭合表。

### 4. 来源真源能够传导到运行候选

统一来源库重建后，正式单字导入和运行刷新可以清除陈旧读音，而不仅是给旧行重算编码。来源排除、
单字库存、音节依据和运行候选已经形成可验证的闭环。

## 当前不能作出的结论

### “只拒绝 11 条”不等于词库已经很干净

来源门禁检查的是注音能否安全、可解释地进入编码链。一个字串有合法拼音，不代表它一定是词，也不
代表它值得静态进入候选框。万象中仍包含人名、地名、诗词、专业条目和联想组合；BCC 中仍包含大量
宽松切分结果。这些应在候选整理覆盖层判断，不应伪装成拼音错误。

### “BCC 高频”不等于“词汇性强”

高频只能决定审查优先级。某些高频项可能来自固定报头、日期模板、连续文本截片或历史语料格式。
必须结合来源分域、长度、上下文、左右边界和组合可恢复性判断。

### “可以动态组合”不等于“应立即删除静态词条”

动态组合必须先通过回放验证。只有当短组件已经批准、读音有来源、组合召回稳定且排序不恶化时，长
候选才可以标为 `dynamic_recoverable`。精简运行词库是路线末端，不是整理起点。

## 分阶段整理路线

### 阶段 A：固定可重复的质量快照

目标：每次来源更新后都能回答“新增、减少和拒绝了什么”。

工作：

1. 重建统一来源包并执行验证；
2. 导出按 BCC 排序的未解码和多读音摘要；
3. 对比 manifest、拒绝数、冲突数和运行候选数；
4. 保证来源库、单字导入和运行刷新形成固定点。

完成标准：相同上游与策略连续重建得到相同统计和摘要；验证为零错误、零警告。

当前状态：基本完成。

### 阶段 B：审查最高频的 517 个未解码字串

目标：建立第一批高价值判断和真实问题分类，不急于自动注音。

至少区分：

- 相对固定的词或固定表达；
- 人名、地名、机构名和历史专名；
- 古代、近代或领域材料；
- 可切分的功能词、助词、连词、量词结构；
- 能由短组件动态恢复的组合；
- 句法片段、偶然组合和明显噪声；
- 证据不足、必须查看上下文的材料。

完成标准：517 项均具有 `approved`、`rejected` 或 `deferred` 决策，并保存依据；不得只填拼音而不判断
候选性质。

### 阶段 C：建立上下文证据和保守规则

目标：把“像不像词”的直觉转为可复查证据。

优先增加：

1. BCC KWIC 前后文；
2. 左右邻接频次、左右熵和互信息；
3. 数字、时间、数量结构和标点边界规则；
4. 助词、语气词、连词、量词及框式结构的切分建议；
5. 万象显式类别到人名、地名、术语等分类的保守映射。

规则只能输出 `proposed` 建议。不能因为命中规则就自动批准，更不能由 LLM 填写拼音或编码。

完成标准：第二批 11,275 个高频项能够按可解释特征分队列，并形成可抽样评估的建议集。

### 阶段 D：整理多读音冲突

目标：把“同形多读”拆成默认读音、限定读音和来源错误，而不是粗暴去重。

审查顺序：

1. 运行高频且存在 BCC 证据的冲突；
2. 单字本调与词内轻声冲突；
3. 人名、地名、姓氏与普通词读音冲突；
4. 文白、古今和方言读音；
5. 低权重孤立来源与可能的注音错误。

完成标准：高频冲突具有明确默认读音或适用范围；无法判断者保持多读或 `deferred`，不猜选。

### 阶段 E：动态组合回放

目标：验证哪些长候选可以由已经批准的短组件恢复。

评测至少包括：

- 目标候选召回率；
- 首屏和前若干位排序；
- 多音组件带来的组合歧义；
- 静态保留与动态恢复的内存、速度和候选质量差异；
- 人名、机构名、框式结构和数量结构等不同类别的分项表现。

完成标准：只有通过回放阈值的条目才能标记为 `dynamic_recoverable`。

### 阶段 F：形成精简发布层

目标：在不破坏来源事实和召回率的前提下缩小静态运行候选。

发布政策：

- 高频基础词、固定表达、重要专名和不能稳定动态恢复的材料静态保留；
- 已验证可恢复的长组合转入动态层；
- 句法截片和噪声不进入运行候选；
- 领域词库可以按场景分包，不必全部常驻；
- 完整来源证据和整理决策继续保留，不随运行精简删除。

完成标准：精简前后具有可比较的回放报告，并能从同一来源真源稳定再生成。

## 每轮整理的固定工作流

```powershell
# 1. 重建统一来源语料包
.\venv312\Scripts\python.exe tools\build_lexicon_source_bundle.py

# 2. 校验来源库
.\venv312\Scripts\python.exe internal_data\pinyin_source_db\validate_source_pinyin_db.py

# 3. 更新可提交的高频审查摘要
.\venv312\Scripts\python.exe tools\export_lexicon_review_summary.py

# 4. 同步完整候选整理全集
.\venv312\Scripts\python.exe tools\build_input_candidate_model.py
```

来源或单字读音发生变化并需要进入运行库时，还应走正式 prototype 单字导入和运行刷新链；不得直接
删除 SQLite 中的个别运行行。

## 近期最合适的下一步

1. 先标注 BCC 频次 ≥10,000 的 517 项；
2. 同时为这些项目提取真实上下文，检验 BCC 高频是否由固定格式或切分噪声造成；
3. 用首批人工决策完善候选分类字段和功能词/数量结构规则；
4. 再处理 1,000–9,999 频次层，形成足以评估规则、统计模型和 LLM 建议质量的标注集；
5. 在完成高频冲突审查和动态组合回放之前，不急于整体削减当前 245 万运行候选。

路线的核心不是给四百多万字串逐一贴上“词/非词”二元标签，而是建立一套能够保存不确定性、区分
来源事实与发布决策、并用动态恢复逐步替代冗余静态组合的输入模型。
