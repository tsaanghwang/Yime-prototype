# 音节编码规则与依据

本文件说明 Yime 如何把来源中的带调拼音变成四个 Yinyuan ID。目标不是证明某套规则永远正确，
而是保证每个现行编码都能回答三个问题：读音从哪里来、经过了什么拼写或兼容变换、最终由哪条正式
编码链生成。

## 权威层次

从高到低依次为：

1. **来源实例**：Unihan 单字读音、phrase-pinyin-data 词语读音，以及经过明确审查的拼音补丁。
2. **拼写与分析规则**：把音节表面形式恢复为编码器使用的首音段和干音段，例如
   `jiu → j + iou`、`yo → y + io`。
3. **编码查表兼容名**：内部干音进入现有码表前的别名归并，例如 `iou → iu`。
4. **音元语义真源**：正式编码器从干音结构生成三个乐音类音元，再和首音组成四个 Yinyuan ID。
5. **布局投影**：Yinyuan ID 到键位；不参与判断一个拼音应由哪些音元组成。

前四层属于语义编码，最后一层才属于键盘布局。

## 来源只证明实际出现的带调音节

现行规则采用“实例驱动”：来源中出现 `yo1`，只能证明 `yo1`；不能由此自动推出
`yo2`、`yo3`、`yo4` 和 `yo5` 都应存在。

早期项目曾使用“发现一个声母—韵母组合后补齐五声”的规则，约生成1836个音节。该规则现在仅以
`LEGACY-FIVE-TONE-CLOSURE` 保存为历史解释，不能再生成现行音节或编码。

外部单字表和词语表在进入本层前先经过[字典拼音第一轮合规审查](DICTIONARY_PINYIN_COMPLIANCE.md)。
审查通过只表示来源形式可安全、可解释地进入工程链，不把罕见字典读音自动提升为普通话核心规范。
`ong` 在此边界先接受声韵分布和零声母拼写检查：`bong4` 虽有来源记录，但作为已知非规范拼式停在
审计层；两条 `wong4` 只有在 U+25948、U+259B7 的字头和字典旁证范围内才校正为 `weng4`，不能扩展
成通用替换。来源中的儿化韵尾省写 `r` 则按《汉语拼音方案》还原为完整编码形式 `er5`。

## 几类容易混淆的形式

### `yo` 与内部干音 `io`

`yo` 在当前单字和词语来源中有真实实例。编码切分把它分析为 `y + io`。这里的 `io` 是有词典实例
支持的内部干音分析；只有来源实际出现的声调才进入现行编码。

### 方案形式与音节拼写形式

`iou/iu`、`uei/ui`、`uen/un` 以及 ü 类省写属于同一形式族中的不同表示层。审计必须同时显示表面
音节和分析干音，但不得把两个写法解释成两套不同音元语义。

### `v` 技术拼音

`v`、`van`、`ve`、`vn` 来自历史输入法或程序数据中用 ASCII 表示 ü 的习惯。它们只能在兼容输入
边界归一化，不能作为规范来源，也不能继续保留一套平行编码。

### 特殊独立音质

`ê`、`m`、`n`、`ng` 及 `hm/hn/hng` 只处理现行来源或经审查补丁实际收录的带调形式。理论上可以
组合出的其他声调不自动进入现行清单。

## 可执行规则目录

[规则目录](../internal_data/syllable_encoding_rule_catalog.json) 为每条规则提供稳定编号、依据、适用层和
处理政策。它被明确禁止保存 Yinyuan ID、码元、键位或 `vk_key`，因此只是解释目录，不是第二套码表。

运行：

```powershell
.\venv312\Scripts\python.exe tools\export_syllable_decomposition.py
```

会生成三张互补表：

- `yime_syllable_decomposition.tsv`：正式编码器实际得到的分段、音元和键位投影。
- `yime_syllable_encoding_provenance.tsv`：每个现行编码的来源计数、规则编号、分析形式和四个 Yinyuan ID。
- `yime_syllable_omissions.tsv`：旧理论集合相对现行实例驱动编码链的差集及其历史原因。

覆盖性测试要求1725个现行编码全部至少有一条来源规则、一条拼写/切分规则和四个正式 Yinyuan ID。
新增音节如果没有登记来源或命中未登记的编码别名，测试必须失败。

## 修改规则的顺序

1. 先核实读音来源及具体声调。
2. 修改来源导入或拼写/切分真源，不从中间码表补入。
3. 运行正式编码器重新生成全部编码和三张审计表。
4. 执行语义测试和布局锁检查。
5. 只有经过独立语义审查，才能更新语义锁摘要；布局改动不得顺带更新摘要。

禁止直接修改 `yinjie_code.json`、四个 Yinyuan ID 或键位来修复单个拼音。
